Общие

[+] Добавить интерфейсную функцию для временного прекращения ( блокировки) обработки заданий указанного типа.  Эта возможность может использоваться, например, для отключения обработки заданий модуля на время его установки.

[+] Возможность остановки работающего задания ( в случае, если прикладное задание периодически проверяет необходимость остановки).

[+] Возможность прерывания выполнения работающего задания с помощью kill session ( интерфейсная процедура AbortTask).

[+] Добавление приоритета и веса для типов заданий, ограничение максимального числа параллельно выполняемых заданий одного типа.

[*] Отказаться от полей %_name_eng и переименовать поля %_name_rus в %_name ( таблицы tp_result, tp_task_status, tp_task_type).

[*] Когда будет приведен к единой версии AccessOperator, убрать динамику в скрипте Install/Schema/Last/tp_task_type.con

Завершение доработок

[-] Удалить таблицу tp_task_log (перестала использоваться для записи лога начиная с версии 2.4.0) и связанные с ней объекты (в т.ч.  pkg_TaskProcessor.body.findTaskLogOld).

[-] Удалить устаревшую (с версии 2.4.0) функцию pkg_TaskProcessor.logMessage (и ее обертки log%) после прекращения использования в прикладных модулях.
