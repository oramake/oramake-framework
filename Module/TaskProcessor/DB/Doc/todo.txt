Title: Доработки

[+] Добавить возможность выполнения указанного ( с помощью task_id) задания
    в текущей сессии ( например, добавив ограничивающий параметр task_id
    для функции <pkg_TaskProcessorHandler.taskHandler>). Данная возможность
    будет удобна при написании тестов в прикладных модулях ( можно создать
    задание без commit и выполнить его в той же сессии).

[+] Добавить возможность управления уровнем логирования при выполнении задания
    ( средствами модуля Logging или добавлением поля с уровнем логирования
    в <tp_task_type>).

[+] Добавить интерфейсную функцию для временного прекращения ( блокировки)
    обработки заданий указанного типа. Эта возможность может использоваться,
    например, для отключения обработки заданий модуля на время его установки.

[+] Возможность остановки работающего задания ( в случае, если прикладное
    задание периодически проверяет необходимость остановки).

[+] Возможность прерывания выполнения работающего задания с помощью
    kill session ( интерфейсная процедура AbortTask).

[+] Проверка корректности команды для обработки ( значение поля exec_command)
    при добавлении/изменении записи в <tp_task_type>.

[+] Добавление приоритета и веса для типов заданий, ограничение максимального
    числа параллельно выполняемых заданий одного типа.

[*] Отказаться от полей %_name_eng и переименовать поля %_name_rus в %_name
    ( таблицы <tp_result>, <tp_task_status>, <tp_task_type>).
