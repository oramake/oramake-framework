Интерфейсные таблицы содержат основные данные по какой-то прикладной системе и используются для предоставления данных по этой системе внешним системам ( обычно создаются для последующей загрузки данных в хранилище).
[!]  Для исключения возможной некорректности данных ( пропуска изменений) после обновлении с помощью материализованного представления добавлена дополнительная проверка дат создания логов на локальных таблицах с исходными данными ( находящихся в той же БД, что и материализованное представление), т.к.
Для регулярного обновления таблицы используется fast-обновляемое материализованное представление с опцией “on prebuilt table” на основе таблицы, которую требуется обновлять.
Обновляет данные таблицы с помощью сравнения содержащихся в ней и актуальных данных и внесения необходимых изменений командами merge и delete ( см.
Является оптимизированным вариантом метода Обновление с помощью сравнения данных: актуальные данные предварительно загружаются во временную таблицу, которая затем используется в командах merge и delete.
Модуль содержит функции для репликации данных, предназначенные для использования в прикладных модулях.
Реализуется с помощью целей сборки модуля из make-целей Установка модуля в БД.
[+] Реализованы методы обновления данных с помощью сравнения данных и с помощью fast-обновляемого материализованного представления.
[+] Добавлен метод обновления данных “Обновление с помощью сравнения данных с использованием временной таблицы” ( см Описание).
[+] Добавлена возможность исключения из обновления некоторых колонок обновляемой таблицы ( указываемых в дополнительной опции excludeColumnList настроек обновления таблицы, формат указан в dsn_data_sync_t.tableList).
[+] Добавлена возможность выдачи прав на использование модуля отдельному пользователю ( ранее права выдавались только для всех пользователей).
[*] По умолчанию при создании логов материализованных представлений с помощью dsn_data_sync_source_t.createMLog на созданные логи стали выдаваться права пользователям, которые имеют права на связанные с этими логами исходные представления ( см.
[*] Для исключения ошибки “ORA-32334: cannot create prebuilt materialized view on a table already referenced by a MV” при создании материализованного представления для обновления таблицы реализовано предварительное удаление зависящих от данной таблицы материализованных представлений ( если они были созданы для обновления интерфейсных таблиц этого же модуля).
[!]  При обновлении с помощью сравнения данных операция delete стала выполняться перед merge для исключения ошибки нарушения уникальности в случае установки значения поля уникального ключа равным значению поля ранее удаленной записи.
