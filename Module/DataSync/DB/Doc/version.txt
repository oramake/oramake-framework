title: Версии



group: 1.3.0 ( 2016-08-26)

[+] Добавлена возможность выдачи прав на использование модуля отдельному
    пользователю ( ранее права выдавались только для всех пользователей).



group: 1.2.1 ( 2015-09-21)

[!] При обновлении с помощью сравнения данных операция delete стала выполняться
    перед merge для исключения ошибки нарушения уникальности в случае установки
    значения поля уникального ключа равным значению поля ранее удаленной записи.



group: 1.2.0 ( 2015-06-30)

[+] Добавлена возможность исключения из обновления некоторых колонок
    обновляемой таблицы ( указываемых в дополнительной опции excludeColumnList
    настроек обновления таблицы, формат указан в <dsn_data_sync_t.tableList>).



group: 1.1.1 ( 2014-10-13)

[*] Для исключения ошибки "ORA-32334: cannot create prebuilt materialized view
    on a table already referenced by a MV" при создании материализованного
    представления для обновления таблицы реализовано предварительное удаление
    зависящих от данной таблицы материализованных представлений ( если они
    были созданы для обновления интерфейсных таблиц этого же модуля).

[*] В процедуру <dsn_data_sync_t.dropRefreshMView> добавлен параметр
    ignoreNotExistsFlag, а также уточнено поведение процедуры
    <dsn_data_sync_source_t.dropMLog> в случае отсутствия лога
    материализованного представления для указанной таблицы.



group: 1.1.0 ( 2014-08-18)

[+] Добавлен метод обновления данных
    "Обновление с помощью сравнения данных с использованием временной таблицы"
    ( см <Описание>).

[*] В процедуре <dsn_data_sync_t.initialize> параметр mlogList стал
    необязательным.

[!] Для исключения возможной некорректности данных ( пропуска изменений) после
    обновлении с помощью материализованного представления добавлена
    дополнительная проверка дат создания логов на локальных таблицах с
    исходными данными ( находящихся в той же БД, что и материализованное
    представление), т.к.  в некоторых случаях Oracle не выбрасывает исключение
    "ORA-12034: materialized view log on ... younger than last refresh"
    при fast-обновлении материализованного представления после пересоздания
    лога.

[*] Добавлены автотесты.



group: 1.0.1 ( 2014-07-02)

[*] По умолчанию при создании логов материализованных представлений с помощью
    <dsn_data_sync_source_t.createMLog> на созданные логи стали выдаваться
    права пользователям, которые имеют права на связанные с этими логами
    исходные представления ( см. параметр grantPrivsFlag). Это позволяет
    обеспечить восстановление прав пользователей на логи материализованных
    представлений после их пересоздания.



group: 1.0.0 ( 2013-10-28)

[+] Реализованы методы обновления данных с помощью сравнения данных и
    с помощью fast-обновляемого материализованного представления.

[+] Реализован объектный интерфейс для упрощения разработки модулей
    интерфейсных таблиц.
