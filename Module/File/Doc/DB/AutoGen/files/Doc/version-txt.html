3.2.0 ( 2014-11-24)

[+] Добавлена возможность получения данных по протоколу HTTP ( проверка наличия ресурса, копирование из, загрузка данных в BLOB или CLOB), в т.ч.  из Интернет ( с использованием прокси-сервера).

[+] Добавлена процедура для перемещения файла ( pkg_FileOrigin.fileMove), которая по возможности выполняется с помощью переименования, что более эффективно, чем последовательное копирование и удаление исходного файла.  В публичных заданиях “Архивирование и удаление файла” ( “arch_del_file”) и “Перемещение файлов в каталог назначения” ( “move_file_in_dir”) стала использоваться процедура перемещения вместо копирования и удаления.

3.1.0 ( 2013-08-23)

[*] Установка общих job-ов для модуля Scheduler.

3.0.3 ( 2013-08-20)

[+] Добавлена процедура pkg_FileOrigin::makeDirectory;

3.0.2 ( 2012-11-28)

[!]  Исправлена ошибка: неверная интерпретация флага перезаписи в процедуре pkg_FileOrigin.unloadClobToFile;

3.0.0.1 ( 2012-11-07)

[!]  Закрытие курсора на чтение из таблицы doc_output_document в java source.

3.0.0 ( 2012-09-28)

[+] Добавлен пакет pkg_FileTest; тестовые скрипты Test/unit-test-megabyte.sql, Test/unit-test-10megabyte.sql, Test/unit-test-100megabyte.sql, Test/unit-test-2gigabyte.sql;

[+] Рефакторинг java-реализации pkg_File в связи с необходимостью добавления считывания файла в определённой кодировке и устареванием некоторых методов java.sql.CLOB, java.sql.BLOB.  Добавлен класс StreamConverter, содержащий статические статические процедуры передачи данных из входных потоков в выходные: StreamConverter::binaryToBinary, StreamConverter::charToBinary, StreamConverter::binaryToChar.  В случае задания кодировки при преобразовании в реализации используются стандартные классы OutputStreamWriter, InputStreamReader для преобразования байтовых потоков в символьные.  При работе с кодировкой БД используются собственные классы WriterOutputStream, ReaderInputStream преобразования символьных потоков в байтовые, содержащие низкоуровневые побайтовые циклы.  Причина: работа с использованием собственных классов примерно в 1.5 раза быстрее, чем работа с использованием стандартных.

[+] Для процедуры pkg_FileOrigin.loadClobFromFile добавлен параметр charEncoding;

[+] Добавлена процедура pkg_FileOrigin.unloadBlobToFile;
