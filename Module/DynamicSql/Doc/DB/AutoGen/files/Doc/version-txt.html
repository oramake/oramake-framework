1.1.2 ( 2018-12-26)

[*] Снято ограничение на длину текста динамического SQL в кэше курсоров dyn_dynamic_sql_t (для текста SQL вместо varchar2 стал использоваться CLOB).

1.1.1 (2017-12-11)

(Установка::1.1.1)

[*] Minor changes in formatting

[+] Ability to group filter conditions (see dyn_dynamic_sql_t.openGroup, dyn_dynamic_sql_t.closeGroup)

[*] Ability to specify a logical operator for a newly created filter condition

[-] Remove call to @oms-drop-type.sql from spec of dyn_dynamic_sql_t, use “create or replace type ... force” instead

[!]  Fix an error causing an infinite loop in getParameterName() when an empty value passed into the conditionText parameter (see dyn_dynamic_sql_t.addCondition)

1.1.0 ( 2012-08-30)

[+] Добавлен функционал для кэширования курсоров пакета dbms_sql ( см.  dyn_cursor_cache_t).

1.0.1( 2011-03-11)

[+] Добавлена процедура dyn_dynamic_sql_t.useCondition, позволяющая использовать несколько групп параметров ( условий) в одном запросе.

[+] Добавлен тестовый скрипт Test/complex-condition.sql, использующий dyn_dynamic_sql_t.useCondition.

1.0.0( 2010-11-10)

[+] Первоначальная версия модуля.
