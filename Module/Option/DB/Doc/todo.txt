Title: Доработки



group: Общие

[+] Для улучшения поддержки списка значений добавить в тип <opt_option_list_t>
    функцию getValueListFlag ( чтобы четко определять, используется ли
    для параметра список значений), а также функцию getListItem для получения
    элементов списка значений в SQL ( аналогично <opt_option_list_t.getValue>).

[+] Для копирования параметров добавить в тип <opt_option_list_t> функцию
    copyOption ( могла был использоваться в модуле Scheduler при копировании
    параметров в случае изменения модуля, к которому относится пакетное
    задание).

[+] Возможно, было бы логично при изменении значения флага
    test_prod_sensitive_flag переносить текущее используемое значение в
    зависимости от типа текущей БД ( в пром. БД переносить общее значение
    в промышленное и наоборот, в тестовых БД переносить общее значение в
    тестовое и наоборот).
