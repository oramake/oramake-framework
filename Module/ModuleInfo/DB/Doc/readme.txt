title: Описание

Модуль предназначен для хранения информации о программных модулях,
установленных в БД, а также о версиях приложений ( application-части
программных модулей), установленных на различные окружения.

*Программный модуль* ( далее *модуль*) на уровне БД представляет собой
совокупность объектов и хранящихся в них данных, решающих определенную
прикладную задачу. В модуль могут также входить дополнительные вспомогательные
скрипты ( например, для выдачи прав на использование модуля). Исходные файлы
модуля храняться в определенной ветке определенного репозитария системы
контроля версий Subversion. С модулем связан неизменный уникальный
идентификатор, состоящий из пути к первоначальному корневому каталогу модуля в
Subversion ( начиная с имени репозитария) и номера правки, в которой был создан
первоначальный каталог модуля. Также для модуля указывается путь к текущему
корневому каталогу в Subversion, который считается уникальным на текущий момент
времени ( он может отличаться от пути к первоначальному каталогу в случае
переименования модуля или переноса его в другой репозитарий).

Модуль может состоять из нескольких частей, одна из которых считается основной.
*Часть модуля* это однозначный и определенный набор исходных файлов и объектов
модуля, которые устанавливаются в одну схему БД и имеют общее назначение в
рамках модуля. Каждая часть модуля имеет собственный ( не пересекающийся с
другими) набор объектов и скриптов. Части модуля могут устанавливаться в разные
базы или разные схемы одной базы ( или даже в одну схему) и взаимодействуют
друг с другом для решения задачи. Части модуля нумеруются начиная с единицы,
первая часть по-умолчанию считается основной.

С текущей функциональностью, реализованной в модуле, связан *номер версии
модуля*, который меняется при выполнении доработок. Номер версии представляет
собой последовательность из нескольких ( обычно трех) неотрицательных целых
чисел, разделяемых точками ( например "2.1.0").  Обычно первое число
увеличивается при внесении принципиальных изменений в модуль, второе число при
выполнении функциональных доработок, третье число при незначительных доработках
или исправлениях, четвертое число используется при выполнении исправлений или
доработок на предыдущих версиях ( например, версия "2.1.2.1", исправляющая
ошибку в версии "2.1.2" при наличии версии "2.1.3" с небольшой доработкой).
*Номер версии части модуля* представляет собой номер версии модуля, в которой в
данную часть в последний раз вносились изменения.

Под установкой модуля в БД понимается загрузка или выполнение в БД файлов
модуля, обеспечивающих создание, удаление или модификацию объектов модуля,
изменение данных модуля, выдачу прав на использование модуля и т.д.

Различаются два типа
установок:
- изменение объектов схемы и данных модуля;
- настройка прав доступа ( включая создание синонимов);

Для первого типа ключевым параметром является схема, в которой расположены
объекты модуля, для второго имя пользователя, чьи права изменяются.

В результате установки сохраняется информация о том, какая часть модуля какой
версии была установлена, а также тип выполненной установки и его ключевые
параметры. Также поддерживается информация о текущих установленных версиях
частей модулей.

Представления для просмотра информации:
- <v_mod_install_module>      - установленные модули ( учитываются только
                                версии объектов схемы основной части модулей)
- <v_mod_install_version>     - текущие установленные версии частей модулей
                                ( в т.ч. версии установок настройки прав
                                доступа)
- <v_mod_install_result>      - информация по результатам всех выполнявшихся
                                установок
- <v_mod_install_action>      - выполнявшиеся действия по установке модулей
- <v_mod_install_file>        - устанавливавшиеся файлы модулей
- <v_mod_install_object>      - устанавливавшиеся объекты БД
- <v_mod_source_file>         - исходные файлы модулей

Информация о установке приложений:
- <v_mod_app_install_version> - текущие версии приложений
- <v_mod_app_install_result>  - информация по результатам всех выполнявшихся
                                установок приложений
