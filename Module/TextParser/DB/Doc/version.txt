title: Версии



group: 1.4.0 ( 2022-02-03)
(<Установка::1.4.0>)

[*] Снятие ограничение длины на поля CSV. Добавление функций getClob.



group: 1.3.5 ( 2016-12-12)
(<Установка::1.3.5>)

[!] В <tpr_csv_iterator_t> исправлена ошибка не позволявшая обращаться по имени
    к столбцам csv-файла с заголовком более 30 символов.



group: 1.3.4 ( 2014-12-03)
(<Установка::1.3.4>)

[+] Для объекта <tpr_csv_iterator_t> добавлена опция noEnclosedCharFlag флаг
    отсутствия специальных символов ограничителей строк ( игнорирование двойных
    кавычек как ограничителей строк).



group: 1.3.3 ( 2013-09-25)

[+] При получении числовых полей из значений ( тип <tpr_csv_iterator_t>)
    предварительно удаляются пробелы для исключения ошибки "character to number
    conversion error".



group: 1.3.2 ( 2013-05-08)

[+] Для функций getDate, getNumber, getString ( тип <tpr_csv_iterator_t>)
    добавлен параметр isNotFoundRaised для возможности получения значения null
    вместо генерации исключения при отсутствии поля с указанным именем.

[+] Для функций getNumber ( тип <tpr_csv_iterator_t>) добавлен параметр
    isTrimPercent для получения значения поля, указанного в виде процентов ( с
    добавлением символа "%").



group: 1.3.1 ( 2013-01-09)

[+] Добавлена функция для проверки наличия поля с указанным именем
    <tpr_csv_iterator_t.isFieldExists>.

[*] В функцию <tpr_csv_iterator_t.getFieldNumber> добавлен параметр
    isNotFoundRaised для возможности получения значения null вместо генерации
    исключения при отсутствии поля с указанным именем.



group: 1.3.0 ( 2012-08-03)

[*] Выдача прав на использование модуля пользователю public



group: 1.2.1 ( 2012-10-15 )
(<Установка::1.2.1>)

[*] <tpr_line_iterator_t>: увеличена максимальная длина строки до 32767 байт.
    Если строка превышает 32767 байт, то для её получения можно использовать
    <tpr_line_iterator_t.getLineClob>.



group: 1.2.0 ( 2012-07-13)

[+] Добавлен пакет <pkg_TextParserTest>. Использован модуль TestUtility.
    Функционал тестового скрипта csv-iterator.sql перенесён в <pkg_TextParserTest>;

[+] Добавлен пакет <pkg_TextParserBase>;

[!] Исправлена ошибка: в случае если файл заканчивается символами разделителя и
    конца файла ( без конца строки), то пустое последнее поле последней строки не
    сохраняется.



group: 1.1.1

[+] При разборе данных в формате CSV с помощью <tpr_csv_iterator_t> добавлена
    возможность задания символа-разделителя полей, что позволяет, например,
    разбирать файлы, сохраненные из MS Excel как
    "Текстовые файлы ( с разделителями табуляции)".

[+] Добавлена функция <tpr_csv_iterator_t::getDataLength>, возвращающая длину
    исходных текстовых данных.

[*] В случае, если в записи с данными меньше полей, чем было в записи с
    именами полей, то значения отсутствующих полей считаются равными null
    ( ранее возникало исключение при попытке получения значения поля)
    ( относится к <tpr_csv_iterator_t>).

[*] Уточнены и дополнены сообщения об ошибках в функциях <tpr_csv_iterator_t>.

[*] В <Описание> добавлен пример использования итератора.



group: 1.1.0

[+] Добавлен объектный тип <tpr_line_iterator_t>, реализующий итератор для
    разбора текста как последовательности строк.
