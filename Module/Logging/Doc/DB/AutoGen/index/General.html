[*] В функции очистки лога pkg_LoggingUtility.clearLog удалять из lg_context_type записи по устаревшим временным типам контекста (на основе значения temporary_use_date).
(скрипт: DB/Example/nested-context.sql)
(скрипт: DB/Example/warn-error-log.sql)
(скрипт: DB/Example/clob-log.sql)

(скрипт: DB/Example/stack-error-log.sql)
Для логирования стека рекомендуется передавать в процедуру raise_application_error в качестве второго параметра результат функции lg_logger_t.errorStack, передавая ей необходимое сообщение.
(скрипт: DB/Example/start-use-log.sql)
[+] Добавить представление (или скрипт) для поиска логов, в которых использовался определенный логер за какой-то недавний период времени.
[!]  В случае, если между выводом отладочных сообщений через dbms_output в одной сессии ( с использованием внутренней процедуры logDebugDbOut пакета pkg_LoggingInternal) прошло более 99 дней, при выводе возникает ошибка “ORA-01873: the leading precision of the interval is too small” при вызове функции pkg_TaskHandler.toSecond.
(скрипт: DB/Example/enabled-level-log.sql)

[*] Выдача прав всем пользователем Install/Grant/Last/all-to-public.sql осуществляется при установке модуля.
[-] Удалены колонки operator_id и триггеры для таблиц lg_destination, lg_level;
[*] Стала использоваться собственная таблица лога для исключения зависимости от модуля Scheduler ( перенесена из Scheduler с минимальными изменениями).

[*] Прекращена поддержка устаревшего иерархического лога на базе поля parent_log_id (использовался в модуле Scheduler до версии 5.1.0), при этом:
[+] Добавлена возможность сохранения в логе полного текста сообщения, указываемого в параметре messageText (тип varchar2, до 32K).
[+] Добавлена возможность установки триггера “after servererror on schema” в пользовательские схемы, иначе при возникновении исключения на клиенте текст сообщения не выводится в расшифрованном виде.
[!]  Исключена ошибка при попытке логирования сообщения длиной свыше 4000 символов ( выбрасывалось исключение “ORA-01461: can bind a LONG value only for insert into a LONG column”, ошибка внесена в версии 1.4.0).
