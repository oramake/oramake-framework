Скрипты

Скрипт представляет собой текстовый файл в кодировке Windows-1251, который может быть синтаксически корректно выполнен с помощью SQL*Plus.

Исходный текст объекта БД должен находиться в скрипте, имя которого совпадает с именем объекта, а расширение соответствует типу объекта.

В результате выполнения скрипта программная единица ( спецификация пакета, тело пакета, процедура и т.д.)  должна пересоздаваться.

В скрипт создания таблицы должены входить DDL-иструкции для создания самой таблицы и связанных с ней объектов ( индексов, ограничений целостности) за исключением триггеров.  Для обеспечения корректного порядка создания объектов ссылочные ограничения ( внешние ключи) таблицы должны находиться в отдельном скрипте ( файл “<имя таблицы>.con”).

Расширения файлов со скриптами

*.conссылочные ограничения ( внешние ключи) таблицы
*.javJava-исходник
*.pksспецификация пакета
*.pkbтело пакета
*.prcпроцедура или функция ( не входящая в пакет)
*.snpматериализованное представление
*.sqsпоследовательность
*.tabтаблица ( включая ограничения целостности и индексы)
*.trgтриггер
*.typSQL-тип ( объявление)
*.tybSQL-тип ( реализация)
*.vwпредставление
*.sqlостальные скрипты

Расширения файлов других типов

*.datфайл с данными для загрузки с помощью SQL*Loader
*.ctlконтрольный файл для SQL*Loader
*.jarJava-библиотека

Файлы для загрузки пакетных заданий ( батчей модуля Scheduler)

*.job.sqlфайл данных задания ( job)
batch.xmlфайл данных пакетного задания
batch_config.xmlфайл настроек пакетного задания ( расписание, опции и т.п.)
