Требования

  • использование последних пропатченных версий клиента Oracle ( поддерживаются версии 11.2.0.2 ( рекомендуемая), а также 11.1.0.7 и 9.2.0.8)

В более ранних версиях встречаются ошибки, позднее устраненные Oracle ( например, ошибки при загрузке файлов с помощью SQL*Loader).

Проверка ( из командной строки)

$ sqlplus -V

SQL*Plus: Release 11.2.0.2.0 Production

Также необходимо проверить наличие и версию утилиты SQL*Loader, которая используется для загрузки больших объемов данных из файлов.

$ sqlldr

SQL*Loader: Release 11.2.0.2.0 - Production on Fri Aug 18 16:08:37 2017

Copyright (c) 1982, 2009, Oracle and/or its affiliates.  All rights reserved.
  • для работы с сетевыми папками следует воспользоваться обходными решениями ( см.  Ошибки::Используемые обходные решения);

Установка для Windows

Порядок установки

  • из командной строки в корневом каталоге модуля выполнить
> .\make install

Пример

> .\make install
installed into: "D:\OraMakeSystem"
( use "make install FORCE=1" to rewrite config files in "D:\OraMakeSystem\usr\local\etc\oms")
  • добавить путь к каталогу установки ( указанному в “installed into: ...”, обычно “D:\OraMakeSystem”) в переменную окружения PATH ( Компьютер -> Свойства -> Дополнительные параметры системы -> Переменные среды)

Проверка корректности установки

  • в новом окне командной строки выполнить ( результат должен быть аналогичен указанному, номер версии может быть больше)
> make --oms-version
make.cmd (OMS) 2.0.0

> oms --version
oms (OMS) 2.0.0
...

Установка для Cygwin

В Cygwin должны быть установлены пакеты

  • bash
  • make
  • coreutils
  • findutils
  • dos2unix
  • gawk
  • sed
  • perl ( для генерации автодокументации)

Порядок установки

  • в терминале Cygwin ( Cygwin Terminal) в корневом каталоге модуля выполнить
$ make install

Пример

$ make install
installed with prefix: /usr/local
( use "make install FORCE=1" to rewrite config files in /usr/local/etc/oms)
  • добавить путь к подкаталогу bin префикса установки ( префикс указан в “installed with prefix: ...”, обычно путь к подкаталогу bin это “/usr/local/bin”) в переменную окружения PATH ( если он не был добавлен в нее ранее);

Проверка корректности установки

  • в новом окна терминала выполнить ( результат должен быть аналогичен указанному, номер версии может быть больше)
$ make --version
GNU Make 3.81
Copyright (C) 2006  Free Software Foundation, Inc.
...

$ oms --version
oms (OMS) 2.0.0
...

Замечания по установке

Замечания

  • после установки может потребоваться отредактировать настроечные файлы ( в .../etc/oms, Настройки баз данных или connect.sh), версии по-умолчанию которых копируются при установке, если данные файлы не существовали.  Настроечные файлы могут быть заменены при использовании флага FORCE=1 при выполнении make install.

Требования к БД при использовании

  • в БД должен быть установлен модуль ModuleInfo для сохранения информации об установке модулей;
  • для возможности загрузки батчей в БД нужно установить модуль Scheduler;
